# 청년·저소득 밀집 지역 분석 기술 문서

## 1. 분석 목적

### 1.1 분석 질문

본 분석은 다음 질문에 답한다.

- 서울시 467개 법정동 가운데 청년인구(20–39세), 1인가구, 기초생활수급 가구가 동시에 밀집한 지역은 어디인가?
- 밀집 정도를 하나의 지수(Concentration Index, CI)로 정량화할 때, 지수 상위 지역은 공간적으로 군집을 이루는가?

### 1.2 전체 프로젝트 내 위치

본 분석은 "서울시 주거비 상승에 따른 청년·저소득 임차가구의 생활권 이탈 위험지역 분석" 프로젝트의 **선행 분석 모듈**에 해당한다. 후속 분석(위험지수 산정)에서 활용할 밀집 지표와 공간 패턴 정보를 생성하는 것이 목적이다.

### 1.3 후속 분석에 제공할 산출물

| 산출물 | 후속 활용 |
| --- | --- |
| `output/seoul_legal_dong_concentration.csv` | 법정동별 CI, 정규화 비율, K-means 군집, LISA 군집 등 전체 분석 결과. 위험지수 산정의 입력 |
| `output/seoul_legal_dong_ci_rank.csv` | CI 순위 요약 |
| `output/seoul_legal_dong_clusters.csv` | K-means 군집별 법정동 목록 |
| `output/seoul_priority_top30.csv` | CI 상위 30개 법정동(우선 관리 대상) |

## 3. 전처리

전처리 코드: `notebooks/preprocessing_legal_dong_risk_base_2024.py`

### 3.1 공통 유틸리티

전처리 전반에서 반복 사용되는 함수들이다.

| 함수명 | 역할 |
| --- | --- |
| `safe_read_csv(path, dtype=str)` | `utf-8-sig` → `utf-8` → `cp949` → `euc-kr` 순서로 인코딩을 시도하여 CSV를 읽는다. `keep_default_na=False`로 빈 문자열을 NaN으로 변환하지 않는다 |
| `normalize_dong_name(value)` | 괄호·특수 구분점 제거, 공백 제거, `홍제제(숫자)` → `홍제(숫자)` 보정, `제(숫자)` 패턴의 `제` 제거 |
| `clean_numeric(series)` | 쉼표·공백 제거, `""`, `"-"`, `"nan"`, `"NaN"` → `"0"` 치환, `pd.to_numeric(errors="coerce").fillna(0)` |
| `normalize_code(series)` | 문자열 변환 후 `.0` 접미사 제거 |
| `safe_ratio(numerator, denominator)` | 분모 0이면 0, 아니면 `numerator / denominator` (`np.where` 사용) |

### 3.2 전처리 단계

#### 단계 1: 행정동-법정동 연결표 전처리 (`preprocess_mapping`)

1. `seoul_legal_dong_information_202506.csv`를 `safe_read_csv`로 읽는다.
2. 필수 7개 컬럼(`시도명`, `시군구명`, `행정동명`, `법정동명`, `행정동코드`, `법정동코드`, `연결번호`) 존재 여부를 검증한다.
3. `시도명 == "서울특별시"`인 행만 필터링한다.
4. 전체 컬럼 기준 중복 행 제거 → `["시군구명", "행정동명", "법정동명", "행정동코드", "법정동코드"]` 기준 중복 행 제거(2단계 중복 제거)한다.
5. 문자열 컬럼(`시도명`, `시군구명`, `행정동명`, `법정동명`)을 `str.strip()`으로 정리한다.
6. 코드 컬럼(`행정동코드`, `법정동코드`, `연결번호`)을 `normalize_code`로 정리한다.
7. `법정동코드`가 `00000`으로 끝나는 서울시/자치구 aggregate 행을 제거해 실제 법정동만 남긴다.
8. `행정동명_정규화`, `법정동명_정규화` 파생 컬럼을 `normalize_dong_name`으로 생성한다.
9. 행정동 기준 법정동 연결 수(`법정동_연결수`): `groupby(["시군구명", "행정동명_정규화"])["법정동코드"].nunique()`로 산출 후 병합한다.
10. 법정동 기준 행정동 연결 수(`행정동_연결수`): `groupby(["시군구명", "법정동코드"])["행정동명_정규화"].nunique()`로 산출 후 병합한다.

- **목적**: 실제 서울 법정동 467개 골격과 행정동-법정동 연결 관계를 확보한다.
- **라이브러리**: `pandas`, `re`

#### 단계 2: 법정동 인구 가중치 전처리 (`preprocess_legal_dong_population`)

1. `legal_dong_population.csv`를 `safe_read_csv`로 읽는다.
2. 필수 컬럼(`법정동코드`, `기준연월`, `시도명`, `시군구명`, `읍면동명`, `계`) 존재 여부를 검증한다.
3. `시도명 == "서울특별시"`이고 `기준연월 == "2024-12-31"`인 행만 필터링한다.
4. `법정동코드`를 `normalize_code`로 정리하고 `법정동코드8 = 법정동코드[:8]`을 생성한다.
5. `계`를 `clean_numeric`으로 변환해 `법정동_인구가중치`를 만든다.
6. `법정동코드8` 기준으로 가중치를 합산한다.

- **목적**: 행정동 단위 count를 법정동으로 나눌 때 사용할 법정동별 총인구 가중치를 만든다. 인구 파일에 없는 12개 법정동은 분할 단계에서 기본 가중치 1을 적용한다.
- **라이브러리**: `pandas`

#### 단계 3: 연령별 인구 데이터 전처리 (`preprocess_population`)

1. `population_by_age.csv`를 `safe_read_csv`로 읽는다.
2. `Unnamed`으로 시작하는 컬럼을 제거한다.
3. 필수 컬럼(`행정구역(동읍면)별`, `5세별`, `항목`, `2024.12 월`) 존재 여부를 검증한다.
4. `항목 == "총인구수[명]"`인 행만 필터링한다.
5. `지역명` 컬럼을 생성하고, 행 단위로 순회하며 현재 자치구(`current_gu`)를 추적하여 `시군구명` 컬럼과 행정동 여부 플래그(`is_admin_dong`)를 할당한다. 서울특별시 25개 구 목록(`SEOUL_GU_LIST`)을 기준으로 구/동 판별한다.
6. 행정동 행만 필터링한다.
7. `행정동명_정규화`를 `normalize_dong_name`으로 생성한다.
8. `연령구간_정규화`를 `5세별` 컬럼에서 공백 제거하여 생성한다.
9. `인구수` 컬럼을 `clean_numeric`으로 `2024.12 월` 값에서 생성한다.
10. 총인구수: `연령구간_정규화 == "계"`인 행을 `groupby(["시군구명", "행정동명", "행정동명_정규화"]).sum()`으로 집계한다.
11. 청년인구수: `연령구간_정규화`가 `{"20-24세", "25-29세", "30-34세", "35-39세"}`에 해당하는 행을 같은 키로 집계한다.
12. 총인구수와 청년인구수를 `outer` 조인으로 병합하고, 결측치를 0으로 채운다.
13. `np.floor`로 소수점 이하를 버리고 `int64`로 변환한다.
14. 총인구수와 청년인구수가 모두 0인 행을 제거한다.
15. 메타 컬럼(`청년연령기준="20-39세"`, `기준연월="2024"`)을 추가한다.

- **목적**: 5세별 연령 인구 데이터에서 행정동별 총인구수와 청년인구수(20-39세)를 산출한다.
- **라이브러리**: `pandas`, `numpy`

#### 단계 4: 가구 데이터 전처리 (`preprocess_households`)

1. `sgis_households_2024_adm_dong.csv`를 `safe_read_csv`로 읽는다.
2. 필수 컬럼(`year`, `sgg_nm`, `adm_nm`, `total_households`, `one_person_households`) 존재 여부를 검증한다.
3. `year == 2024`인 행만 필터링한다(`clean_numeric`으로 연도 비교).
4. 컬럼명을 한글로 변환한다: `sgg_nm` → `시군구명`, `adm_nm` → `행정동명`, `total_households` → `총가구수`, `one_person_households` → `1인가구수`.
5. `시군구명`, `행정동명`을 `str.strip()`으로 정리한다.
6. `행정동명`이 `["합계", "소계", "기타"]`인 행을 제거한다.
7. `행정동명_정규화`를 `normalize_dong_name`으로 생성한다.
8. `총가구수`, `1인가구수`를 `clean_numeric` → `np.floor` → `int64`로 변환한다.
9. `기준연도 = 2024`를 추가한다.
10. `groupby(["시군구명", "행정동명", "행정동명_정규화", "기준연도"]).sum()`으로 집계한다.

- **목적**: SGIS 가구 데이터에서 행정동별 총가구수와 1인가구수를 산출한다.
- **라이브러리**: `pandas`, `numpy`

#### 단계 5: 기초생활수급 가구 데이터 전처리 (`preprocess_basic_livelihood`)

1. `number_of_basic_livelihood_recipient.csv`를 `safe_read_csv`로 읽는다.
2. 필수 컬럼(`자치구`, `행정동`, `총 수급가구_2024`) 존재 여부를 검증한다.
3. `자치구 == "합계"`인 행을 제거한다.
4. `행정동`이 `["소계", "기타"]`인 행을 제거한다.
5. 컬럼명을 변환한다: `자치구` → `시군구명`, `행정동` → `행정동명`, `총 수급가구_2024` → `기초수급가구수`.
6. `시군구명`, `행정동명`을 `str.strip()`으로 정리한다.
7. `행정동명_정규화`를 `normalize_dong_name`으로 생성한다.
8. `기초수급가구수`를 `clean_numeric` → `np.floor` → `int64`로 변환한다.
9. `기준연도 = 2024`를 추가한다.
10. `groupby(["시군구명", "행정동명", "행정동명_정규화", "기준연도"]).sum()`으로 집계한다.

- **목적**: 행정동별 기초생활수급 가구수를 산출한다.
- **라이브러리**: `pandas`, `numpy`

#### 단계 6: 행정동 → 법정동 인구비례 분할 (`allocate_admin_values_to_legal_dong`)

각 데이터셋(population, households, basic_livelihood)에 대해 동일 함수가 호출된다.

1. 행정동 단위 데이터를 `["시군구명", "행정동명_정규화"]` 키로 연결표와 `left` 조인한다(`indicator=True`).
2. 병합 실패(`left_only`) 행을 분리하여 기록한다.
3. 연결된 각 법정동에 `legal_dong_population.csv`의 `법정동_인구가중치`를 붙인다.
4. 인구 파일에 없는 법정동은 기본 가중치 1을 적용한다.
5. 행정동별 가중치 합계(`행정동_가중치합계`)를 계산하고, 각 값 컬럼에 대해 `np.floor(값 × 법정동_분할가중치 / 행정동_가중치합계)`로 인구비례 분할한다(소수점 이하 버림).
6. `groupby(["시도명", "시군구명", "법정동명", "법정동코드"]).sum()`으로 법정동 단위 집계한다.
7. 원본 합계와 분할 후 합계의 차이·차이율·버림값·누락 가중치 법정동 수·병합실패 건수를 검산 레코드로 생성한다.
8. 차이율이 1% 이상이면 경고를 출력한다.

- **목적**: 행정동 단위 인구·가구 count를 법정동 기준으로 변환한다. 하나의 행정동이 여러 법정동에 걸칠 때, 각 법정동의 총인구 비중만큼 할당한다.
- **라이브러리**: `pandas`, `numpy`

#### 단계 7: 법정동 기준 최종 데이터셋 병합 (`build_final_dataset`)

1. `make_legal_dong_base`: 연결표에서 법정동 골격 테이블(`시도명`, `시군구명`, `법정동명`, `법정동코드`, `연결된_행정동수`)을 생성한다. `["시군구명", "법정동코드"]` 기준 중복 제거.
2. `merge_allocated_dataset`: 골격 테이블에 인구 분할 결과(`총인구수`, `청년인구수`), 가구 분할 결과(`총가구수`, `1인가구수`), 수급 분할 결과(`기초수급가구수`)를 순차적으로 `left` 조인한다. 병합 후 결측치는 `clean_numeric` → `np.floor` → `int64`로 0 처리한다.
3. 파생 비율 변수를 `safe_ratio`로 산출한다:
   - `청년비율 = 청년인구수 / 총인구수`
   - `1인가구비율 = 1인가구수 / 총가구수`
   - `기초수급가구비율 = 기초수급가구수 / 총가구수`
4. 메타 컬럼(`기준연월="2024"`, `청년연령기준="20-39세"`)을 추가한다.
5. 컬럼 순서를 정리하고 `["시군구명", "법정동코드", "법정동명"]`으로 정렬한다.

- **산출물**: `data/processed/legal_dong_risk_base_2024.csv`
- **라이브러리**: `pandas`, `numpy`

#### 단계 8: 품질 점검 (`run_quality_checks`)

1. 법정동코드 중복 수를 확인한다.
2. 전체 컬럼 결측치 수를 확인한다.
3. 총인구수·총가구수·기초수급가구수에 음수 값이 있는지 확인한다.
4. 비율 변수(`청년비율`, `1인가구비율`, `기초수급가구비율`)가 [0, 1] 범위인지 확인한다.
5. 총인구수·청년인구수·총가구수·1인가구수·기초수급가구수가 모두 0인 행 수를 확인한다.
6. 각 데이터셋별 병합 실패 행정동 수를 확인한다.
7. 원본 합계와 법정동 분할 후 합계의 차이를 출력한다.

#### 단계 9: 검산 결과 저장

- `output/legal_dong_allocation_check.csv`: 데이터셋별 분할 전후 합계 차이

## 5. 분석 방법

분석 코드: `notebooks/analyze_legal_dong_concentration_2024.py` (실행 스크립트)
분석 로직: `notebooks/analysis_modules/` 패키지 (모듈별 분리)

### 5.1 데이터 로드 및 검증

1. **입력 데이터 로드** (`analysis_modules/data.py: load_base_dataset`)
   - `data/processed/legal_dong_risk_base_2024.csv`를 `pd.read_csv(encoding="utf-8-sig", dtype={"법정동코드": str})`로 읽는다.
   - 필수 12개 컬럼 존재 여부를 검증한다.
   - 수치 컬럼 8개를 `pd.to_numeric(errors="coerce").fillna(0)`으로 변환한다.
   - 총인구수 0인 법정동도 포함해 전처리 결과 전체를 분석 대상으로 사용한다.
   - `법정동코드8 = 법정동코드[:8]`을 생성한다.
   - 분석 대상이 정확히 467개인지, `법정동코드8`에 중복이 없는지 검증한다.

2. **GeoJSON 로드** (`analysis_modules/data.py: load_or_download_geojson`)
   - `data/raw/seoul_neighborhoods_geo_simple_2015.geojson`이 존재하면 로드한다. 없으면 `requests.get`으로 `southkorea/seoul-maps` 저장소에서 다운로드한다.

3. **경계 매칭 검증** (`analysis_modules/data.py: validate_geojson_match`)
   - GeoJSON의 `EMD_CD` 집합과 데이터의 `법정동코드8` 집합이 정확히 일치하는지 검증한다. 불일치 시 `ValueError`를 발생시킨다.

### 5.2 밀집지수(Concentration Index) 산출

**기법**: 가중합 복합지수 (Min-Max 정규화 → 가중 합산)

함수: `analysis_modules/scoring.py: calculate_concentration_index`

1. **Min-Max 정규화** (`minmax_normalize`): 각 비율 변수를 `(x - min) / (max - min)`으로 [0, 1] 범위로 변환한다. `min == max`이면 0을 반환한다.
   - `청년비율` → `청년비율_norm`
   - `1인가구비율` → `1인가구비율_norm`
   - `기초수급가구비율` → `기초수급가구비율_norm`

2. **가중합** (`weighted_sum`): 가중치 합이 1인지 검증한 후 가중 합산한다.
   - 기본 가중치(`DEFAULT_WEIGHTS`): `청년비율_norm=0.35`, `1인가구비율_norm=0.30`, `기초수급가구비율_norm=0.35`

3. **순위 산출**: `pd.Series.rank(ascending=False, method="min")`으로 CI 내림차순 순위를 부여한다.

### 5.3 K-means 군집 분석

**기법**: K-means 군집 분석 (k=4, `sklearn.cluster.KMeans`)

함수: `analysis_modules/scoring.py: run_kmeans_clustering`, `assign_cluster_labels`

1. **표준화**: `sklearn.preprocessing.StandardScaler`로 `["청년비율", "1인가구비율", "기초수급가구비율"]` 3개 변수를 z-score 표준화한다.
2. **군집화**: `KMeans(n_clusters=4, random_state=42, n_init=20).fit_predict()`로 4개 군집을 할당한다.
3. **라벨 할당** (`assign_cluster_labels`): 군집 중심의 비율 평균값을 기준으로 자동 라벨링한다.
   - `청년비율`이 최대인 군집 → `"청년 밀집형"`
   - 나머지 중 `기초수급가구비율`이 최대인 군집 → `"저소득 밀집형"`
   - 나머지 중 `concentration_index`가 최소인 군집 → `"일반형"`
   - 나머지 → `"복합 밀집형"`

### 5.4 공간 자기상관 분석 (Moran's I / LISA)

**기법**: Global Moran's I, Local Moran's I (LISA) with Queen contiguity

함수: `analysis_modules/spatial.py: build_queen_neighbors`, `calculate_moran_lisa`

1. **Queen 인접행렬 구축** (`build_queen_neighbors`):
   - GeoJSON의 Polygon/MultiPolygon 꼭짓점을 `precision=8` 소수점 자리로 반올림한다.
   - 동일 꼭짓점을 공유하는 법정동을 이웃으로 판별한다(Queen contiguity: 꼭짓점 하나라도 공유하면 이웃).
   - 결과: `dict[str, set[str]]` 형태의 인접 관계.

2. **Global Moran's I** (`calculate_moran_lisa`):
   - `value_col = "concentration_index"`
   - `z = values - mean(values)` (평균 중심화)
   - `I = (n / S0) × (Σᵢ Σⱼ wᵢⱼ zᵢ zⱼ) / (Σᵢ zᵢ²)`
   - `S0` = 전체 이웃 링크 수 (가중치는 이진, row-standardize하지 않음)

3. **Local Moran's I (LISA)**:
   - `spatial_lag_i = mean(z[neighbors])` (이웃의 z 평균)
   - `local_i = z_i × spatial_lag_i`
   - **순열 검정**: `permutations=999`, `seed=42`. 매 순열에서 `z`를 무작위 셔플하고 local I를 재계산한다. 유사 p값 = `(|permuted_local_i| >= |observed_local_i|의 횟수 + 1) / (permutations + 1)`.
   - **LISA 군집 분류** (p ≤ 0.05일 때):
     - `z_i > 0` & `lag_i > 0` → `"HH"` (High-High, 핫스팟)
     - `z_i < 0` & `lag_i < 0` → `"LL"` (Low-Low, 콜드스팟)
     - `z_i > 0` & `lag_i < 0` → `"HL"` (공간적 이질)
     - `z_i < 0` & `lag_i > 0` → `"LH"` (공간적 이질)
     - 그 외 또는 p > 0.05 → `"Not significant"`

### 5.5 우선순위 선정

함수: `analysis_modules/scoring.py: build_top30`

1. **TOP30 선정** (`build_top30`): 기본 가중치 CI의 `concentration_rank` 기준 상위 30개 법정동을 추출하고, `is_lisa_hotspot_hh = (lisa_cluster == "HH")`를 추가하며, `priority_rank`(1–30)를 부여한다.

### 7.8 종합 해석

1. **밀집의 이중 구조**: 서울의 청년·저소득 밀집은 단일한 현상이 아니다. "대학가·역세권 중심의 청년 밀집"과 "도심 노후 주거지 중심의 저소득 밀집"이라는 두 가지 구조적으로 다른 취약성이 공존한다. 인구비례 분할 후 TOP30에서는 저소득 밀집형이 더 많이 포함된다.

2. **도심권 핫스팟 집중**: HH 핫스팟의 88.4%가 종로·중구·영등포 3개 자치구에 집중한다. 밀집 취약 법정동은 고립된 점이 아니라 연속된 권역을 형성하고 있어, 개별 동 단위가 아닌 **권역 단위의 정책 접근**이 필요함을 시사한다.

3. **법정동 인구비례 분할의 효과**: 균등분할에서 같은 행정동에 연결된 법정동들이 동일한 값과 CI를 갖는 문제가 줄었다. 새 결과에서는 CI 고유값이 439개/467개로 늘어 법정동 간 차이가 더 잘 드러난다.

4. **1인가구비율의 매개 역할**: 1인가구비율은 청년비율(r=0.83)·기초수급비율(r=0.39) 양쪽과 양의 상관을 가진다. 청년 밀집과 저소득 밀집이라는 서로 다른 유형의 취약성에 1인가구가 공통적으로 높게 나타나므로, 1인가구비율은 두 유형을 관통하는 공통 취약 요인으로 해석할 수 있다.
